1.- Entornos Virtuales:

Creación: python -m venv nombre_entorno
OJO: git no avisa que se creo el entorno virtual
verificar si se creo con ls o ir directo a la carpeta

Activación: 
	- Windows - Powershell : nombre_entorno\Scripts\activate.bat
	- Unix/MacOS: source nombre_entorno/bin/activate
	- Windows - Gitbash: source nombre_entorno/Scripts/Activate

Desactivación: deactivate

pip list --> Listar librerias instaladas
pip install nombre_libreria --> Instalar librerías
pip uninstall nombre_libreria --> Desinstalar librerías

* crear archivo de requerimientos: pip freeze > requirements.txt
* instalar librerias desde el archivo de requerimientos(requirements.txt): pip install -r requirements.txt


2.- Instalar Django
- Windows - Powershell : py -m pip install Django==6.0.2
- Unix/MacOS/Linux: python -m pip install Django==6.0.2
- Windows - Gitbash: *opción 1 indicando versión a instalar: pip install Django==6.0.2 
		     *opción 2 instalación con la última versión: pip install Django

- Verificar la versión instalada usar pip list o python -m django --version

3.- Creación de un Nuevo Proyecto en Django

opción 1: django-admin startproject nombre_sitio nombre_mi_proyecto
opción 2: django-admin startproject nombre_mi_proyecto
opción 3: django-admin startproject nombre_mi_proyecto .

Probar corriendo el servidor: opción 1: python manage.py runserver
			      opción 2: python manage.py runserver 6300 --> Aquí indicamos el puerto en el que queremos que corra

4.- Creación de una aplicación dentro de nuestro proyecto
paso 1: en consola "cd nombre_mi_proyecto" a la carpeta del proyecto (verificar con ls si esta el archivo manage.py)
paso 2: python manage.py startapp nombre_aplicacion


5.- Registrar la app creada en settings.py
Abre en visual studio el proyecto completo (carpeta), luego ve a nombre_mi_proyecto/settings.py 
y en lista INSTALLED_APPS agrega tu aplicación (nombre_aplicacion):

INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'nombre_aplicacion',   
]

7.- Creamos el archivo de urls de nuestra aplicación en la carpeta de la aplicación "nombre_aplicacion"  (urls.py)

from django.urls import path

from . import views

urlpatterns = [
    path("", views.index, name="index"),
    path("home", views.home, name="home"), ---> Otro ejemplo
]

8.- Modificamos el archivo urls.py de nuestro proyecto e incluimos el nuevo archivo urls.py que hemos 
creado en nuestra aplicación, deberia quedar así: (agregamos al import el include y el path de nuestra app)

from django.contrib import admin
from django.urls import include, path

urlpatterns = [
    path("my_app", include("nombre_aplicacion.urls")), ----> EJEMPLO ---> 127.0.0.1:8000/my_app/home
    path("admin/", admin.site.urls),
]
