# Clave primaria (Primary Key) en una base de datos relacional

Una **clave primaria** (*primary key*) es un **campo** o **conjunto de campos** que identifica de forma **única e inequívoca** cada registro (**fila**) dentro de una tabla.

Su objetivo es **garantizar la integridad de los datos** al **prohibir valores duplicados y nulos**, asegurando que cada fila sea distinguible.

---

## Características y reglas principales

- **Unicidad:** no puede haber dos filas con el mismo valor en la clave primaria.
- **No nulo:** no puede contener valores `NULL`.
- **Única por tabla:** solo se puede definir **una** clave primaria por tabla.
- **Inmutabilidad (recomendación):** se recomienda que sus valores **no cambien** con el tiempo.

---

## Ejemplos comunes

- **Tabla `Clientes`:** `ID_Cliente` (numérico)
- **Tabla `Productos`:** `SKU` o `Codigo_Producto`
- **Tabla `Empleados`:** `Numero_Seguridad_Social`

---

## Clave primaria compuesta

Si no existe una sola columna que sea única, se puede usar una **combinación de dos o más columnas** para crear una **clave primaria compuesta**.

Esto asegura la **unicidad del conjunto**, aunque los valores individuales se repitan.

---

## Importancia

- Permite una **rápida recuperación** de datos.
- Es esencial para establecer relaciones con **claves foráneas** (*foreign keys*) en otras tablas.
